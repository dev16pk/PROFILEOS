# Explainosphere

**Universal Curiosity Learning Platform** - Search any topic, get curated learning packages with videos, articles, quizzes, and projects. From tech to finance, science to engineering.

## 🎯 Problem We Solve

Ever searched for a term online and ended up with 50+ tabs, half-baked understanding, and no clear path forward? Explainosphere solves the **infinite tab syndrome** by providing:

- ✅ Structured learning packages (not just text answers)
- ✅ Intent-based depth (curious vs. deep dive vs. complete mastery)
- ✅ Multi-source aggregation (videos, articles, code examples)
- ✅ Mind-map exploration (navigate without getting lost)
- ✅ Validation tools (quizzes and projects)

## 🏗️ Architecture

### Tech Stack

**Backend:**
- FastAPI (Python)
- PostgreSQL (database)
- Redis (caching)
- OpenAI GPT-4 (content synthesis)
- YouTube Data API (video scraping)

**Frontend:**
- Next.js 14 (React framework)
- TypeScript
- TailwindCSS
- React Flow (mind-map visualization)

### System Flow

```
User Query → Intent Detection → Cache Check → [Scrape + Synthesize] → Return Package → Cache
```

## 🚀 Quick Start

### Prerequisites

- Python 3.10+
- Node.js 18+
- PostgreSQL 14+
- Redis (optional but recommended)
- OpenAI API key
- YouTube Data API key (optional for MVP)

### Backend Setup

1. **Navigate to backend directory:**
```bash
cd backend
```

2. **Create virtual environment:**
```bash
python -m venv venv
venv\Scripts\activate  # Windows
# or
source venv/bin/activate  # Mac/Linux
```

3. **Install dependencies:**
```bash
pip install -r requirements.txt
```

4. **Set up environment variables:**
```bash
copy .env.example .env
# Edit .env and add your OpenAI API key
```

5. **Initialize database:**
```bash
# Create PostgreSQL database
psql -U postgres
CREATE DATABASE explainosphere;
\q

# Run schema
psql -U postgres -d explainosphere -f ../database/schema.sql
```

6. **Run the server:**
```bash
python main.py
```

Backend will be running at `http://localhost:8000`

### Frontend Setup

1. **Navigate to frontend directory:**
```bash
cd frontend
```

2. **Install dependencies:**
```bash
npm install
```

3. **Set up environment:**
```bash
# .env.local is already created with default values
# Edit if your backend runs on a different port
```

4. **Run development server:**
```bash
npm run dev
```

Frontend will be running at `http://localhost:3000`

## 🔧 Configuration

### Backend (.env)
```env
OPENAI_API_KEY=your_key_here
DATABASE_URL=postgresql://user:pass@localhost/explainosphere
REDIS_URL=redis://localhost:6379
```

### Frontend (.env.local)
```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## 📖 API Documentation

Once the backend is running, visit:
- Swagger UI: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`

### Key Endpoints

**Search:**
```
POST /api/search/
Body: {"query": "What is RAG?", "intent_override": "curious"}
```

**Generate Quiz:**
```
POST /api/search/{query_id}/generate-quiz
```

**Track Progress:**
```
POST /api/progress/
Body: {"topic": "RAG", "status": "exploring"}
```

## 🚢 Deployment

### Backend (Railway/Render)
1. Connect GitHub repo
2. Set environment variables
3. Deploy from main branch

### Frontend (Vercel)
1. Connect GitHub repo
2. Set `NEXT_PUBLIC_API_URL`
3. Deploy automatically

### Database (Supabase)
1. Create project
2. Run schema.sql
3. Update DATABASE_URL
